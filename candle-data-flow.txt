Candle Data Fetching Flow in TradeNotify
=====================================

1. User opens the chart page
- The user visits the chart route in the frontend.
- The page component ChartPage renders the ChartContainer component.

2. Frontend creates the chart UI
- In Frontend/src/components/ChartContainer.jsx, the app creates a Lightweight Charts instance.
- It also creates a candlestick series to display the price data.

3. Frontend starts loading candle data
- The ChartContainer component runs a function named fetchData().
- This function is triggered when the chart component loads.

4. Frontend sends an API request
- The frontend uses the Axios instance from Frontend/src/api/axios.js.
- It calls the backend endpoint:
  /market/candles?symbol=...&interval=...

5. Backend route receives the request
- The request reaches the controller in Backend/src/controllers/market.controller.js.
- The getMarketCandles() function reads the query parameters:
  - symbol
  - interval

6. Backend validates the input
- The controller checks if the symbol exists.
- It also validates the interval using the mapping in Backend/src/config/market.config.js.
- If the interval is valid, it determines a default range for the request.

7. Backend calls the market data service
- The controller then calls:
  marketDataService.getCandles(symbol, interval, range)
- The service is implemented in Backend/src/services/marketData.service.js.

8. Backend service calls Yahoo Finance
- The service builds a request URL like:
  https://query1.finance.yahoo.com/v8/finance/chart/{symbol}?interval={interval}&range={range}
- It uses axios to fetch the data from Yahoo Finance.

9. Yahoo Finance returns raw chart data
- Yahoo Finance responds with chart data containing timestamps and OHLC values.
- The service reads the raw data from the response.

10. Backend transforms raw data into candles
- The service extracts:
  - timestamp
  - open
  - high
  - low
  - close
- It loops through the records and creates a cleaned array of candle objects.
- It also removes invalid or null values.

11. Backend sends formatted candles back to frontend
- The controller wraps the candle data in a JSON response:
  {
    success: true,
    data: {
      symbol,
      interval,
      candles
    }
  }
- This response is sent back to the frontend over HTTP.

12. Frontend receives the response
- The frontend gets the response in ChartContainer.jsx.
- It checks whether the response is successful and whether data.candles is an array.

13. Frontend cleans and sorts the candle data
- The received candles are filtered to remove malformed entries.
- They are sorted by time so the chart can display them correctly.

14. Frontend renders the candles on the chart
- The frontend passes the cleaned data into Lightweight Charts using:
  candlestickSeries.setData(cleanData)
- The chart then displays the candles for the selected symbol and time interval.

Important note about live updates
- This flow is for historical candle data.
- It does not keep polling every 5 minutes automatically.
- It fetches once when the chart loads and again only if the symbol or interval changes.

Additional stock lookup flow
- The project also has a separate stock list flow for search and symbol selection.
- The backend loads stock metadata from Backend/src/config/stocks.json.
- The controller in Backend/src/controllers/stock.controller.js reads this JSON file and exposes endpoints for:
  - GET /api/stocks
  - GET /api/stocks/search
- The search endpoint filters stock records by symbol or company name using the JSON array directly.
- It does not search a database; it searches the loaded JSON data in memory.
- The frontend search UI in Frontend/src/components/SearchBar.jsx calls these endpoints and shows results in the search panel.
- By default, the UI does not show the entire JSON file to the user; it requests a limited subset (around 100 results) for the search/list view.

Timezone and market-time behavior
- The candle timestamps returned by the backend are numeric Unix timestamps from Yahoo Finance.
- The frontend chart in Frontend/src/components/ChartContainer.jsx passes these timestamps directly to Lightweight Charts.
- The chart labels and tooltips are formatted using the Asia/Kolkata timezone so they appear in Indian time.
- There is no explicit hardcoded logic in the frontend that says the market is only open from 09:15 to 15:30.
- In this codebase, the chart simply displays the candle timestamps provided by the backend and formats them for Indian time display.

Summary of data flow
Frontend Chart -> Frontend API Client -> Backend Controller -> Backend Market Service -> Yahoo Finance -> Backend Service -> Backend Controller -> Frontend -> Lightweight Charts
